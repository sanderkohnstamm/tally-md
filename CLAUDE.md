# Tally.md — Markdown Todo Editor

A three-pane desktop markdown editor for managing tasks with a done log, built with Tauri 2.x.

The flow: **Todo → Today → Done**. Pick what to work on today, then mark items done. Done items accumulate as a log organized by date — a record of what you accomplished.

## Tech Stack

- **Backend:** Rust + Tauri 2.x (serde, chrono, dirs, git2, keyring)
- **Frontend:** Vanilla JS + CodeMirror 6 (no framework)
- **Bundler:** esbuild
- **Package manager:** npm

## Project Structure

```
src-tauri/          Tauri/Rust backend
  src/main.rs       Entry point, IPC command handlers
  src/finished.rs   Completion/recovery logic, breadcrumb building, date headers
  src/git_sync.rs   Git clone/pull/push/sync via libgit2
  src/settings.rs   Settings struct, load/save to JSON (local + repo)
  tauri.conf.json   Tauri config
src-ui/             Frontend source
  main.js           Core app logic, CodeMirror setup, keybindings, IPC calls
  themes.js         8 color theme palettes
ui/                 Built frontend (served by Tauri)
  index.html        HTML template + settings/shortcuts overlays
  style.css         Stylesheet
  bundle.js         Generated by esbuild
src/                Original TUI source (legacy, not used by desktop app)
```

## Data Storage

- **Local mode:** files in configurable path (default `~/.todos/`)
- **Git mode:** files in `~/.tallymd/repos/<repo-name>/`, synced via git
- Files: `todo.md`, `today.md`, `done.md`, `settings.json`

## Commands

- `npm install` — install JS dependencies
- `npm run dev` — start Tauri dev mode
- `npm run build` — bundle frontend JS
- `npm run watch` — watch frontend for changes
- `npm run package` — build production binaries

## Architecture

Tauri IPC commands:
1. `load_files()` — reads all three files, fills missing date headers in done.md
2. `save_files(todo, today, done)` — writes all three to disk
3. `complete_item(source, target, cursor_line, to_done)` — moves item forward (todo→today or today→done with breadcrumb)
4. `recover_item(source, target, cursor_line, from_done)` — moves item backward
5. `reformat_dates(done_content, new_format)` — rewrite date headers when format changes
6. `load_settings()` / `save_settings(...)` — settings persistence
7. `git_init_repo` / `git_pull` / `git_push` / `git_sync_full` — git operations
8. `git_store_token` / `git_has_token` / `git_delete_token` — keychain token management

Frontend uses three CodeMirror 6 editors with dynamic keybindings, auto-save (1s debounce), 8 themes, heading fold/unfold, clickable links, and resizable panes. All keyboard shortcuts are customizable via settings.
